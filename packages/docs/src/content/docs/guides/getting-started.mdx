Lilybird <u>does not</u> officially support <u>**deno**</u>. While the core should work, there's no guarantee that the modules or, for that matter the examples will.

## Installation

The fastest way to get started with Lilybird is by using our template generator.

<Tabs>
  <TabItem label="bun" icon="bun">
    ```bash frame="none"
    bun create @lilybird
    ```
  </TabItem>
  <TabItem label="npm" icon="seti:npm">
    ```bash frame="none"
    npm create @lilybird
    ```
  </TabItem>
  <TabItem label="pnpm" icon="pnpm">
    ```bash frame="none"
    pnpm create @lilybird
    ```
  </TabItem>
  <TabItem label="yarn" icon="seti:yarn">
    ```bash frame="none"
    yarn create @lilybird
    ```
  </TabItem>
</Tabs>

## Running your first bot

After running the command you should see a file structure somewhat like this:

:::note
Depending on the selected language for the template the structure may differ: The JavaScript template will not contain type or config files (`globals.d.ts` and `tsconfig.json`)
:::

<FileTree>

- globals.d.ts
- tsconfig.json
- package.json
- .env \<- add your **token** here
- src
  - index.ts

</FileTree>

### Adding your token

To add your bot token open your `.env` file and add the token after `TOKEN=`.

### Starting your bot

:::tip
If you are using Bun you can run `bun start` no matter what language you are using.
:::

<Tabs>
  <TabItem label="TypeScript">
    ```bash frame="none"
    npm run dev
    ```
  </TabItem>
  <TabItem label="JavaScript">
    ```bash frame="none"
    npm run start
    ```
  </TabItem>
</Tabs>

### Ready for production

If you are using TypeScript we highly recommend adding `NODE_ENV="production"` to your `.env` file and if you are using Node, do the following.

:::note
If you are using <u>**Bun**</u> we highly encourage changing or adding a script to run production using `bun --smol`.
:::

```bash frame="none"
npm run build
npm run start
```
